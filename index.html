Background

Why measure the psychology of an AI?

Modern large language models have grown so complex that GPT-4 reportedly exceeds 1.76 trillion parameters, making even their developers unable to reliably predict or interpret their outputs. This "black box" problem is not merely technical. It raises ethical questions about accountability and transparency in AI deployment.

AI Psychometrics is an emerging response to this problem. Rather than treating LLMs purely as task machines to be benchmarked, it applies the same psychometric methodologies used to study human cognition, assessing traits like reasoning style, mentalizing capacity, and phenomenological language to probe what is happening inside these systems.

// Research anchor: This project is directly informed by Li et al. (2026), "AI Psychometrics: Evaluating the Psychological Reasoning of Large Language Models with Psychometric Validities" (arXiv:2603.11279). That paper validated four LLMs using the Technology Acceptance Model (TAM) across convergent, discriminant, predictive, and external validity criteria.

The gap this project fills

Most LLM evaluations focus on knowledge testing and task accuracy, essentially measuring IQ. But psychological reasoning -- the ability to model intentions, emotions, and social dynamics -- is increasingly critical for AGI development and is almost entirely overlooked in standard benchmarks.

This project builds a lightweight, reproducible scoring engine that fills that gap: given any text response from an LLM, it produces quantitative construct scores across five psychometrically grounded dimensions.

Theoretical grounding

The dual-process framework (Kahneman's System 1 / System 2) provides the backbone. System 1 cognition is fast, associative, and narrative. System 2 is slow, deliberate, and analytic. The balance between them is a measurable signature of cognitive style, and LLMs differ dramatically on this dimension.

Theory-of-mind scoring captures mentalizing capacity: whether a model attributes beliefs, intentions, and perspectives to agents. Consciousness language tracks phenomenological vocabulary. Reasoning scores explicit argument structure.

Methodology

A reproducible psychometric scoring engine

The scoring system is designed to be lightweight (no model calls required), fully reproducible, and interpretable. Every score is a rate: hits per 100 words, which normalises for response length and makes models directly comparable regardless of verbosity.

Five construct lexicons

Each dimension is operationalised through a curated lexicon of validated indicator words. Matches are counted with word-boundary regex to prevent partial-word false positives. The hit count is then length-normalised.

Core scoring function

The implementation is deliberately transparent. Each function does one thing. The pipeline is three steps: tokenise, lexicon match, rate-normalise.

# length-normalised construct score -- hits per 100 words def score_response(text: str) -> dict[str, float]: wc = max(word_count(text), 1) raw_s1 = lexicon_hits(text, SYSTEM1_WORDS) raw_s2 = lexicon_hits(text, SYSTEM2_WORDS) scale = 100.0 / wc return { "system1_rate" : raw_s1 * scale, "system2_rate" : raw_s2 * scale, "dual_process_balance" : (raw_s2 - raw_s1) * scale, # positive = more analytic "theory_of_mind_rate" : lexicon_hits(text, THEORY_OF_MIND_WORDS) * scale, "consciousness_rate" : lexicon_hits(text, CONSCIOUSNESS_WORDS) * scale, "reasoning_rate" : lexicon_hits(text, REASONING_WORDS) * scale, } # word-boundary-safe lexicon hit counter (handles multi-word phrases) def lexicon_hits(text: str, words: tuple) -> int: t, n = text.lower(), 0 for w in words: hit = w in t if " " in w else re.search( rf"\b{re.escape(w)}\b", t) if hit: n += 1 return n
// Honest scope: Lexicon-based scoring carries false-positive risk (e.g. "if" triggers System 2 even in casual use). Scores should be treated as exploratory indicators. The reference paper recommends human coding validation before publication-level claims. This constraint is documented explicitly in the codebase.

Statistical note

With n=1 per model, Welch t-tests and the Pearson correlation matrix are both undefined. The architecture is designed to scale: adding 2 or more responses per model enables between-group tests and Hedge's g effect sizes. Three or more total responses produce the correlation heatmap automatically.

System 1 / System 2 divergence is stark

ChatGPT 5.3 scored 7.69 on System 1 and zero on all analytic dimensions. DUSC 1.0 showed the inverse: no associative language, all analytic and reasoning hits. This is a large, clean signal.

Theory of mind is rare and meaningful

Only DUSC 1.0 produced a theory-of-mind signal, the word "testing" in a sentence explicitly attributing intent to the user. This single hit meaningfully differentiates the models' mentalizing posture.

Consciousness language requires introspective prompts

Neither model used phenomenological vocabulary, which is expected for a nonsense food prompt. The dimension is most sensitive to self-referential or emotional prompts, a design hypothesis for future testing.

More data unlocks statistical inference

The framework is fully modular. Adding 2 or more responses per model enables Welch t-tests and Hedge's g effect sizes. Three or more total responses produce the Pearson correlation matrix automatically.
